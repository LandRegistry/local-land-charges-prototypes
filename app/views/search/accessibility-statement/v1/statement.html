How you can use this website

On this website you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 200% without the text spilling off the screen
  • navigate most of the website using only a keyboard
  • navigate most of the website using speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS (Job Access With Speech), NVDA (NonVisual Desktop Access) and Voiceover

We have made the website text as simple as possible to understand, aiming for a reading age of a 12-year-old. We only use technical terms where there is no easier wording to use without changing the meaning.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

We run accessibility tests when a feature is new or changed. This means most of this website is fully accessible.

We know the following parts are not accessible:

  • users with keyboard-only control cannot access all features of the map
  • some screen readers and browsers may repeat, miss, or misread text on the page
  • HTML errors flagged on certain pages
  • some pages do not handle certain kinds of error messages

How to get information in an accessible format

We are always looking to improve the accessibility of this website.

You can contact us:

  • about any problems accessing information
  • to request information in a different format, such as audio recordings, braille or an accessible PDF
  • if you find any problems that are not listed on this page
  • to tell us how we are not meeting accessibility requirements

How we tested this website

We tested this website on 1 July 2020. This includes our PDF documents, available as a download through Official Searches.

Our internal audit team take care of all testing. Tools we use are:

  • internal testing against the W3C WAI Web Content Accessibility Guidelines version 2.1
  • logging all results and defects in Quality Center
  • WAVE evaluation to check for contrast detection, heading, legend and label checks
  • W3C Markup Validation Service (for HTML validation)
  • Web Developer devices to disable JavaScript and check table layout, links, field sets and images with alt text
  • Dragonspeak and JAWS screen readers
  • PAC3 (PDF Accessibility Checker 3) automated devices, to assess PDF accessibility compliance

What we are doing to improve

When we change or add a new feature the website goes through accessibility testing. If we identify any issues, we will aim to fix them as soon as possible.

Area management, Restrictions, Regulation zones and Reporting units

This theme covers areas under an environmental policy or activity impacting the environment.

Policies include:

  • reducing pollution levels
  • conserving natural resources
  • protecting plant, animal, and human health
  • controlling development

Local land charges which form part of this theme group include smoke control orders.

Land use

This theme covers the use and functions of a territory and its description of land. This depends on its socio-economic and ecological purpose.

The 2 types of land use are:

  • The existing land use, which outlines the use and functions as they are now
  • The planned land use, defined by spatial planning authorities to outline ways to use the land

Local land charges which form part of this theme group include Legal agreements as to the use of land, and Tree Preservation Orders.

Protected sites

This theme focuses on the specific conservation of designated areas of land or sea. It protects and maintains:

  • listed buildings
  • biological diversity
  • conservation areas

Local land charges which form part of these theme group include Ancient Monuments and Sites of Special Scientific Interest.
